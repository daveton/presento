# Presento - 设计文档

## 项目概述

**Presento** = 一键把视频变成可直接讲的 PPT

用户粘贴视频链接或文案 → 点击生成 → 获得可直接使用的 PPT 文件。

---

## 核心原则

1. **单一入口**：只有一个输入框
2. **单一结果**：只输出 PPT
3. **零选择**：用户不做任何选项决策
4. **可直接用**：下载的 PPT 不需要大改

---

## 技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | Next.js 14 + TailwindCSS + shadcn/ui | 快速开发、SEO友好、组件丰富 |
| 后端 | FastAPI (Python) | 异步支持、类型安全、AI生态好 |
| AI | GPT-4 (OpenAI API) | 结构化输出稳定 |
| PPT | python-pptx | 生成真实 .pptx 文件 |
| 部署 | Vercel (前端) + 云服务器 (后端) | 成本可控 |

## 限制条件

### 输入限制
- 视频链接：支持 YouTube、Bilibili（URL 解析）
- 纯文本：用户直接粘贴文案
- 长度限制：视频 ≤ 30 分钟，文本 ≤ 5000 字

### 生成限制
- 请求模式：**同步**（先同步，超时再升级异步）
- 超时：30 秒
- 算力：每次生成调用 1 次 GPT-4
- PPT 页数：≤ 10 页

### PPT 约束（规则引擎）
```
每页标题：≤ 22 个中文字符
每页要点：2-5 条
每条长度：≤ 20 个中文字符
总页数：≤ 10 页
字号范围：20pt - 28pt（根据数量动态调整）
```

## 系统架构

```
Frontend (Next.js)
    │ POST /api/generate {input: string}
    ▼
Backend (FastAPI)
    │
    ├─ Step 1: 解析输入
    │   ├─ 如果是 URL → 提取文案（或提示用户粘贴）
    │   └─ 如果是纯文本 → 直接使用
    │
    ├─ Step 2: LLM 生成结构化内容
    │   ├─ Prompt: 强制 JSON 输出
    │   └─ Output: {"title": "", "slides": [{"title": "", "points": []}]}
    │
    ├─ Step 3: 规则引擎处理
    │   ├─ 分析每页内容长度
    │   ├─ 匹配最佳布局（cover/bullet/two_column）
    │   ├─ 应用约束（截断/分页/字号调整）
    │   └─ 生成布局配置
    │
    └─ Step 4: 渲染 PPT
        ├─ 使用 python-pptx
        ├─ 填充内容到对应布局
        └─ 返回 .pptx 文件 URL
    │
    ▼
Frontend
    └─ 展示预览 + 下载按钮
```

## 核心模块设计

### 1. 布局定义（代码写死，MVP 版）

```python
LAYOUTS = {
    "cover": {
        "type": "TITLE_ONLY",
        "title_max": 22,
        "subtitle_max": 40,
    },
    "bullet_3": {
        "type": "TITLE_BULLET",
        "points_min": 2,
        "points_max": 3,
        "point_max_len": 20,
        "font_size": 28,
    },
    "bullet_5": {
        "type": "TITLE_BULLET", 
        "points_min": 4,
        "points_max": 5,
        "point_max_len": 18,
        "font_size": 24,
    },
    "two_column": {
        "type": "TWO_COLUMN",
        "points_min": 6,
        "points_max": 8,
        "point_max_len": 16,
        "font_size": 20,
    }
}
```

### 2. 规则引擎优先级

```
P0 (绝不妥协):
  1. 标题 > 22 字 → 智能截断
  2. 要点 > 5 条 → 自动分页
  3. 单条 > 20 字 → 强制换行/截断

P1 (动态调整):
  4. 2-3 条 → 大字号 (28pt)
  5. 4-5 条 → 中字号 (24pt)
  6. 6-8 条 → 双列 + 小字号 (20pt)

P2 (体验优化):
  7. 每页预留 20% 留白
  8. 行间距 1.2-1.5 倍
```

### 3. LLM Prompt 结构

```
System:
你是一位专业的 PPT 内容设计师。根据用户提供的文案，
设计一个可直接演讲的 PPT 结构。

约束（必须遵守）：
1. 标题 ≤ 22 个中文字符
2. 每页要点 2-5 条
3. 每条 ≤ 20 个中文字符
4. 总页数 ≤ 10 页
5. 输出必须是 JSON，不要任何解释

输出格式：
{
  "title": "PPT主标题",
  "slides": [
    {
      "type": "cover",
      "title": "封面标题",
      "subtitle": "副标题"
    },
    {
      "type": "content",
      "title": "页面标题",
      "points": ["要点1", "要点2", "要点3"]
    }
  ]
}

User: {input_text}
```

## 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| LLM 超时 | 中 | 高 | 先同步，超时转异步队列 |
| 链接解析失败 | 高 | 中 | 首页文案提示"或粘贴文案" |
| 输出格式不稳定 | 低 | 高 | 强制 JSON + 重试机制 |
| PPT 排版溢出 | 中 | 中 | 规则引擎兜底截断 |
